7  // This example illustrates how to make a custom virtual controller for
8  // the purpose of feeding custom input into InControl.
9  //
10  // A virtual device is necessary because InControl is device centric internally and
11  // this allows custom input to interact naturally with other devices, whether it be
12  // joysticks, touch controls, or whatever else. Custom input cannot be "force fed"
13  // into other existing devices. A device is considered authoritative over the input
14  // it provides and cannot be directly overriden. However, by creating your own
15  // virtual device, you can provide whatever input you desire and you gain all the
16  // benefits of being a first class device within InControl.
17  //
18  // For more advanced situations you may want to have a device manager to organize
19  // multiple devices. For an example of how to accomplish this,
20  // see XInputDeviceManager and XInputDevice.
21  //
